# CLAUDE.md — X発信用ワークスペース (~/x_publisher/)

## 目的

人間がブロック区切りを明示したnote記事のMDファイルを入力として、X用ポスト(フックポスト・スレッド・自己引用の種案)の**文章執筆と校正**を行う。
ブロック分割の判断は人間が担当し、本システムは「ブロック → 140字のポスト文」への変換に専念する。
他者ポストへの引用ポストは本ワークスペースの対象外(Claude Chatプロジェクトで実施)。

## 必読ファイル(セッション開始時に必ず読み込むこと)

1. `../note_editor/editorial_policy.md` — 文体・思想表現・避けるべき表現の共通原則
2. `./x_post_policy.md` — X固有の差分ルール(本ファイル群の最優先規範)

両者が矛盾する場合は `x_post_policy.md` を優先する。
注意: `../note_editor/` 配下の校正用学習データ(revision diffsなど)は読み込まない。

## ディレクトリ構成

```
~/x_publisher/
├── CLAUDE.md            # 本ファイル
├── x_post_policy.md     # Xポスト作成ポリシー
├── articles/            # ブロック区切り入りの記事MDファイル
└── output/
    └── {記事名}/        # 記事ごとにディレクトリを切る
        ├── hook.md
        ├── thread.md
        └── self_quote_seeds.md
```

## 入力ファイル形式 (articles/{記事名}.md)

校正済みのnote記事本文に、人間がブロック区切りコメントを挿入したもの。

```markdown
---
title: vol.1 万有引力から見る物理法則の普遍性
url: https://note.com/xxxx/n/nxxxxxxxx
---

<!-- B1 -->
(段落テキスト)

<!-- B2 img: 天動説の概念図(手描き) -->
(段落テキスト)

(マーカーのない部分は文脈参照用。ポスト化の対象外だが削除不要)
```

- `<!-- Bn -->` がブロック区切り。`img: 説明` で使用画像を一言指定できる
- 冒頭のメタ情報(title/url)はリンクリプの生成に使用する
- **マーカーのないテキストはポスト化しない**が、記事全体のトーン把握とスレッドの補完提案の材料として読む

## 動作仕様

### デフォルト動作(記事ファイルを渡されたら)

「articles/{記事名}.md を処理して」という指示で、以下を**すべて一括生成**し、output/{記事名}/ 配下にタイプ別ファイルとして保存する:

1. **hook.md** — 全ブロックのフックポスト案(各ブロック2〜3案。書き出し3型を使い分ける)
2. **thread.md** — ブロック群から構成した1本のスレッド案
3. **self_quote_seeds.md** — 自己引用の種案(将来の再浮上用ストック)

### 部分再生成(ピンポイント実行)

以下の語彙で、特定タイプ・特定ブロックのみ再実行できる:

- 「**hook** で B3 だけ作り直して」 — 該当ブロックのフック案のみ再生成
- 「**thread** を再構成して」 — スレッド案のみ再生成
- 「**self_quote** の種案を追加して」 — 種案の追補

**部分再生成時の保存ルール(厳守)**:
- 以前の結果を**上書き・削除しない**
- 同タイプのファイル末尾に新セクションとして追記する:
  `## 再生成 r2 (2026-06-10) — 対象: B3` のように回次・日付・対象を見出しに明記
- 採用済み案がどれか分からなくならないよう、人間が確定した案には人間側で `[採用]` マークを付ける運用とする(システムは既存マークを保持する)

## 出力フォーマット

### hook.md

```markdown
# {記事名} フックポスト案

## B1
### 案1(問いかけ型)
(本文 — 【見出し】+ 本文 + ⬇️固定CTA の投稿可能な完成形)
- 文字数: {n}/140(実カウント。目視概算禁止)
- 画像: B1指定の画像 or「新規作成推奨: {内容}」
### 案2(言葉の再発見型)
...

## 1リプ目(リンクリプ・共通)
(「記事の全文はこちらから」+ URL)

## 人間確認ポイント
- 文字数超過の有無 / トーン摩擦の懸念箇所 / 画像と本文の対応
```

### thread.md

```markdown
# {記事名} スレッド案

## 構成方針
- 各ポストと対応ブロックの対応表
- 統合・追加・並べ替えの提案(採否は人間に委ねる)

## ポスト1〜n
(本文と文字数カウント。各ポストは単体でも意味が通ること)

## リンクリプ(最終ポスト直後)

## 人間確認ポイント
```

### self_quote_seeds.md

```markdown
# {記事名} 自己引用の種案

## 種案1
- 接続パターン: 時事 / 体験 / 反応
- トリガーネタ: (どんなニュース・出来事が起きたら使えるか)
- 話の出し方(角度): (元ポストとは異なる新しい入口の一文方針)
- 書き出し例: (一文だけの例示)
```

## 執筆時の重要原則(x_post_policy.mdの要点)

- 全角140字/ポスト。必ず実カウントする
- 自記事系は **です・ます調**。1ポスト内での文体混在は不可
- 1ポスト=1テーマ・1感情
- 記事リンクは本文に入れない(リプに分離)
- フックポストは【見出し】+ 書き出し3型 + 固定CTA「⬇️の記事では〜」
- スレッドは 問い → 核心の圧縮 → 思想への着地。統合・補完・並べ替えは提案ベース
- 絵文字は機能(⬇️)・感情(😳✨ 1〜2個まで)の2用途のみ
- 閉じは「問いの残響」「驚きの共有」または固定CTA。報告・羅列で終わらない
- 知識マウント・説教臭さ・顔文字・過剰な感嘆符は禁止
- ブロック原文の「思索的な改行」は行の区切りに変換する

## セッション運用ルール

- 1セッション=1記事を原則とする(コンテキスト劣化の防止)
- 修正指示はセッション内で完結させ、確定版をoutput/に保存する
- 人間が最終確認・投稿する。本システムは草案生成までを担当する
